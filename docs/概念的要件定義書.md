# 概念的要件定義書：ProjectCodeMap

## 1. 目的と哲学
本ツール「ProjectCodeMap」は、AI駆動開発（AIDD）において、LLM（大規模言語モデル）に対して**「最小限のトークン消費で、正確なコードベースの構造・コンテキストを伝達する」**ことを目的とする。

LLMが大規模なコードベース全体を正確に理解するためには、全文を読み込ませるのではなく、ツールの解析結果に基づく「地図（Map）」を共有することが最も効率的であるという方針をとる。

## 2. 実施・実装の方針 (Scope)
「何を、何で、どう実装するか」

*   **信頼性重視のAST解析 (tree-sitter採用):** 
    *   単純な正規表現による抽出は行わない。プログラミング言語の文法に基づいた正確なAST解析を採用し、関数定義やクラス構造を確実に抽出する。
*   **制御の柔軟性:**
    *   ユーザーがAIに渡す範囲を完全制御できるよう、`.gitignore` および `.pcmignore` を最優先で尊重する仕組みを実装する。
*   **LLM最適化出力:**
    *   AIエージェントの種類（Aider、Cursor、純粋なLLMチャット等）に合わせて、最適なフォーマット（Text, JSON, Markdown, XML）へ変換可能な抽象化されたフォーマッタ構造を実装する。

## 3. 実装しない方針 (Non-Scope)
「何を、何で、どう実装しないか」

*   **コード内容の全量読込:**
    *   効率化を最優先するため、ファイル内の全コードを解析結果に含めることは行わない。あくまで構造、シグネチャ、ドキュメントの抽出に留める（トークン消費の爆発を防ぐため）。
*   **ツール内部へのAI統合:**
    *   本ツールは「LLMへコンテキストを渡すためのツール」であり、それ自体にLLMの推論エンジンや複雑なAI機能を内蔵させることはしない。ツールの目的を「正確な情報抽出」に純化させる。
*   **GUI/複雑なUI:**
    *   CLIツールとして実装する。GUIはコンテキストを複雑にし、移植性や統合効率を落とすため、GUIやリッチなUIは実装しない。
